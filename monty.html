The Problem

This is a simulation of the famous Monty Hall Problem. Here's the deal:
  1. You're playing a game show where you can potentially win a brand new car.
  2. The host offers you three doors: Door 1, Door 2, and Door 3.
  3. You pick one door, but don't open it yet.
  4. Your host then opens one of the losing doors (these doors commonly hold something that most normal people don't really want, like a goat).
  5. You are then given the option to stay on your current door or switch to the remaining other closed door.
So what's the bettter option? Should you stay? Should you switch? Does it matter?

The Simulation

This simulation runs the Monty Hall problem either once ('Run Once!') or a thousand times ('Run a Thousand Times!'). You can change between 'Switch' mode, where the simulated player switches after the host reveals a losing door, and 'Stay' mode, where the simulated player holds his/her ground. Here's a small breakdown of how it works:
    Simulation Procedure
  1. The simulation randomly selects 1, 2 or 3 as the 'winning door'.
  2. The simulation randomly picks a door for the simulated user to 'pick'. This again is 1, 2 or 3, and is independent of the 'winning door' pick
  3. Finally, the simulation finds a door that is neither the correct door nor the door selected by the simulated user. This represents the door opened by the host
  4. An array is then built that holds the remaining, closed doors (i.e., minus the host door).
  5. If the user has selected switch mode the simulated user switches doors. Otherwise, the simulated user stays.
  6. Finally, the simulation compares the simulated user's final guess with correct door. Results are stored in an array
